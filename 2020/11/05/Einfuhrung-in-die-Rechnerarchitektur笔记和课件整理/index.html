Negative Ganze Zahlen

方案一: 用一个二进制数表示+/-

  • 比如负数的最高位是1: -1=1000 0001
  • 缺点: 计算复杂
  • 缺点: 0有两种表述形式: 1000 0000=0000 0000=0

 

大多使用的方案二: 二的补数(Zweierkomplement)

  • 一个数的负数等于这个数取反+1
  • 例子: -1 = 取反(Inv)(0000 0001) +1 = 1111 1110 +1 = 1111 1111
  • -1 + 1 = 1111 1111 + 0000 0001 = 1 0000 0000 -> 0000 0000 = 0
  • 适用于任何数

 

Nicht vorgeschrieben -> gewählte Konvention=>Entwurfsentscheidung beim Rechnerdesign!

Ausrichtung der Daten im Speicher

数据结构对齐的定义(data alignment)

  • 一个数据是和n-字节限制对齐的, 如果他的地址A是n的整数倍: A mod n = 0

 

常见的要求:

  • s-Byte的数据形式必须和s-字节限制对齐

  • 背景是: 储存器是用字(Worten)来组织的

    • 根据数据宽度(总是和架构宽度一致)传输一次访问, 比如32 Bit或者64 Bit
    • 对对齐的对象来说数据访问次数最小

 

Nichtausgerichtete Daten (misaligned data)

在一些处理器里面是允许的

  • 半字, 字或者双字允许储存在任意字节地址

 

优点

  • 在混合不同的数据形式时可以无缝使用储存器(不存在padding(填充)操作)
  • 对老的ISA有更好的兼容性(Kompatibilität)

 

缺点

  • 需要额外的内存访问
  • 更高的硬件需求

ERA022-System-Befehle Privilegierte Modi

 

Herausforderungen in Mehrbenutzersystemen

起初系统是只为一个用户设计的

  • 一个时间点只有一个程序
  • 没有网络和远端的访问

 

如今的系统大多经常是多用户系统

  • 很多不相关的程序在储存器中
  • 可能同时执行

 

先决条件

  • 两个程序之间的数据保护
  • 程序间切换的可能

这些都是操作系统的任务!

 

需要来自硬件的支持

  • 有优先级的空间, 在这个空间里面操作系统工作
  • 中断/错误指示
  • 储存器管理

 

Unterstützung von Privilegierter Ausführung

如今的计算机大多是多用户系统

  • 用户程序不应该被允许直接访问通用硬件资源和系统配置
  • 需要通过操作系统编码
  • 技术基础: 执行空间

 

系统空间(Systemmodus)

  • 对所有计算机部分的完全的(优先的)访问
  • 给操作系统的

 

用户空间(Benutzermodus)

  • 被限制的访问
  • 没有优先的指令比如输入和输出
  • 无法访问系统配置

 

现代的处理器还可能有多个空间

  • 大多是用户和操作系统无法看见的
  • 只能硬件访问

 

Betriebssystem-Unterstützung: Moduswechsel

从系统空间到用户空间

  • 优先指令, 通常: 改写控制寄存器
  • 操作系统把这个用在程序的开头

 

从用户空间到系统空间

  • 必须被监控
  • 不能再系统空间里执行任意代码

 

通过特殊的指令实现

  • 也被称作: 系统调用, 巴拉巴拉的
    • 确保回跳地址, 跳到操作系统中预定义好的地址, 转入系统空间
    • 回跳指令再转回用户空间
    • 对比: 操作系统的明确定义的子程序

  • 系统调用指令会引发一个特殊的例外

 

中断/例外也能导致切换到优先(系统)空间

 

ERA023-System-Befehle Interrupts/Exceptions/Traps

 

Unterbrechungen und Ausnahmen(Interrupts und Exceptions/Traps)

冯诺依曼规定了顺序的执行

  • 程序计数器再每一条指令执行完后加一
  • 有一些例外: 比如跳跃
  • 对大多程序的单一执行是足够了

 

现代的系统更加复杂

  • 多用户系统
  • 复杂的外部设备
  • 互联网连接
  • 交互性

 

附加的: 错误处理

  • 比如: 除以0或者访问一个不存在的储存器
  • 理想状态: 在程序中处理

 

于是我们就引入了中断(Interrupt)和例外/陷阱(Exceptions/Traps)

 

Unterbrechungen („Interrupts“)

允许处理器对外部的, 非同步的事件作出反应

  • 特殊的输入信号: 中断请求
  • 可以遮盖(Maskierung)(处理)终端或者忽视(Ignorierung)

 

步骤(流程):

  • 信号在指令循环的尾部被询问
    • 可能是具体的外部信号线

  • 如果有信号

    • 把程序计数器保存在栈中
    • 跳到之前在操作系统里预定义好的地址(Unterbrechungsbehandlungsroutine, 中断处理例程 interrupt handler)
    • 回跳到原先的执行位置的尾部

  • 对比: 调用子程序, 但是不同步

 

主要应用:

  • 输入/输出
  • 时间计数器的运行

 

Ausnahmen („Traps“ / „Exceptions“)

执行不允许的操作会导致例外

  • 英语翻译, 自己看看

 

一些例子:

  • 不被允许的指令
  • 算术错误, 比如除以0
  • 储存器访问错误
    • 主要储存器里面没有对应的地址
    • 错误的访问权限
    • 硬件问题
  • 在硬件里面没有实现的指令

 

共同之处:

  • 被程序里的操作引发

 

Ausnahmebehandlung („Trap“ / „Exception“)

处理器中的过程

  • 中断刚才执行的指令
  • 保证运行状态(特别是程序计数器)
  • 跳到系统空间之前预定义的地址去

 

„Exception“-Vektortabelle

  • 存储了对于不同种类的例外的处理例程

 

操作系统中对例外的可能的处理

  • 停止处理
  • 仿真操作?(Emulieren der Operation)
  • 给用户查错

 

如果好有意义的话: 重复或者继续执行被中断的指令

 

Interrupts vs. Exceptions/Traps

两者都导致中断

  • 停止当前的程序
  • 储存系统状态(比如程序计数器)
  • 操作系统接过控制权
  • 结束后重返程序

 

中断是不同步的

  • 被外部事件触发
  • 可以在任意指令后发生
  • 中断大多情况下都是正常的运行

 

例外/陷阱是同步的

  • 在加工指令的时候触发
  • 在这条指令后发生
  • 通常情况下是执行中错误的信号

Mehrbenutzersysteme

操作系统在过程之间来回切换

  • 中断程序执行
    • 比如: 通过时间控制的中断
  • 把老的程序计数器和所有的寄存器储存(Code 1)

Phasen eines Compilers

词汇分析(Lexikalische Analyse)

  • 向象征表填入类型信息
  • 检查程序的语义逻辑连接性(特别是类型匹配)

ERA039-Mikroprogramme

Idee der Mikroprogrammierung

每个ISA指令通过一个微程序实现

  • ISA指令被表述成一个操作代码
    • 编码的功能
    • 编码的操作对象
  • 操作代码必须能触发一个微程序的执行

 

宏和微程序的接触点

  • 操作引导模块通过ISA(宏)操作码
  • ISA/宏指令寄存器也是微架构的一部分

 

微架构并不确定ISA, 但是确定了(基础)指令形式

  • 和ISA紧密结合
  • 在例子中: 8 bit指令+8 bit寄存器
  • 更长的形式可以在微程序中实现

 

Einbindung des Makroinstruktionsregister

 

Adressbildung durch Opcodes

 

Makro-Mikro Opcode Abbildungsspeicher

一个宏/ISA指令通过一个微程序表达

  • 也可以多个指令
  • 额外的: MSP地址0被其他东西占了
    • 还是机器的执行

 

结果: 1:1的映射没有意义

 

映射储存器

  • 把ISA的操作代码映射成微程序地址
  • 定义了哪些操作通过哪些指令展示
  • 是ISA的定义的中心

 

通过表格实现

  • 每个ISA操作代码有一个
  • 经常和MSP是同一个技术

 

Mikroprogrammspeicher

微指令必须为了调用先准备好

  • 分开的微程序储存器, 可寻址
  • 在微架构里面
  • 相对较少的地址, 但是很宽

Einzelne Mikroinstruktionen

很长的Bit数组带有专门的空间为了

  • 1 Bit的信号, 直接控制元素
  • 很多bit的信号
  • 常数

 

选择: 微程序的编码展示法

  • 不是单个信号的控制, 而是信号组
  • 减少微程序的储存器需求
  • 对编程者更加友好
  • 缺点: 在MIR和硬件控制点的解码需要时间, 通过硬件实现

 

Mikroinstruktionsformat

Mikroprogrammzyklus

  1. 确定下一个微指令的地址
  2. 把这条指令从微程序储存器里面选出来
  3. 把这条指令存入MIR-一个带有一些bit位的信号缓存器, 控制整个系统的信号
  4. 信号触发所有元素里面的操作

 

执行收到时间片的驱使

  • 外部的时间片(时钟信号)
  • 控制控制信号什么时候是有效的和持续的时间
  • 被信号传播来限制
    • 转换装置和逻辑电路需要时间来接收/处理信号
    • 传播了很远的信号经过了很多电路了
