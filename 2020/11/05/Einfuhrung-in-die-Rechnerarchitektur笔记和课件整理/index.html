ERA003 Datendarstellung

计算机(一般)在二进制系统下工作

  • 通过0/1来描述
  • 通电/没通电(分别代表了对应二进制的两种状态)

结果:

  • 数据通过一串0/1来表示

 

Zahlensysteme

  • 我们习惯了10进制
  • 其他文化还有其他的进制,比如玛雅人用的20进制
  • 对于计算机来说, 二进制是比较好的
  • 二进制太长难以阅读, 于是就有了16进制, 优点: 每一位16进制代表4个二进制的数

 

Binär- und Hexadezimal-Arithmetik

  • 在其他进制的算数和十进制的规则是一样的.

  • 对于二进制, 加法只有8种可能, 所以可以通过列表来表述

 

Gängige Wortlängen

对于未定义的字长(Wortlängen)很难去操控

  • 数据转换长度
  • 储存空间
  • 计算

所以计算机系统通常都有一套处理好的字长.(见下表)

Negative Ganze Zahlen

方案一: 用一个二进制数表示+/-

  • 比如负数的最高位是1: -1=1000 0001
  • 缺点: 计算复杂
  • 缺点: 0有两种表述形式: 1000 0000=0000 0000=0

 

大多使用的方案二: 二的补数(Zweierkomplement)

  • 一个数的负数等于这个数取反+1
  • 例子: -1 = 取反(Inv)(0000 0001) +1 = 1111 1110 +1 = 1111 1111
  • -1 + 1 = 1111 1111 + 0000 0001 = 1 0000 0000 -> 0000 0000 = 0
  • 适用于任何数

 

Nicht vorgeschrieben -> gewählte Konvention=>Entwurfsentscheidung beim Rechnerdesign!

 

有了负数之后字长的取值范围就有了一些变化,但是字长能代表的数量不变.

 

Darstellungen von Realen Zahlen

二进制表示原则上被限制在了整数范围

  • 负数已经是个问题了
  • 需要一个特殊的实数的表示方式

 

方案一: 固定小数点表示法

  • 一个两个部分的元组
    • 小数点前的数
    • 小数点后的数
  • 通过这两个部分的组合来表现实数

在数值上有问题

  • 被限制的范围
  • 多余的精度

 

Fließkommazahlen

(这是方案二)

为了更加灵活: 把一个数分成底数M(Mantisse)和指数(Exponent) ($M$*$basis^E$)

  • 大多数系统都遵守IEEE-754
  • 优点: 表示范围很广
  • 缺点: 分配不均匀

Warnung: alle Formate haben nur eine endliche Genauigkeit

 

Text

Textzeichen müssen kodiert werden!

字符必须被编码(kodiert ).

 

Programm-Abstraktion

  • C/C++和Fortran可以直接转成汇编语言
  • Java先转成Byte Code, 然后通过虚拟机转成汇编语言
  • Python/Skript和Mathlab/R Programm要通过翻译器才能转成汇编语言
