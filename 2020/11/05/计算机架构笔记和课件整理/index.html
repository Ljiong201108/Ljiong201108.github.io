Negative Ganze Zahlen

方案一: 用一个二进制数表示+/-

  • 比如负数的最高位是1: -1=1000 0001
  • 缺点: 计算复杂
  • 缺点: 0有两种表述形式: 1000 0000=0000 0000=0

 

大多使用的方案二: 二的补数(Zweierkomplement)

  • 一个数的负数等于这个数取反+1
  • 例子: -1 = 取反(Inv)(0000 0001) +1 = 1111 1110 +1 = 1111 1111
  • -1 + 1 = 1111 1111 + 0000 0001 = 1 0000 0000 -> 0000 0000 = 0
  • 适用于任何数

 

Nicht vorgeschrieben -> gewählte Konvention=>Entwurfsentscheidung beim Rechnerdesign!

Ausrichtung der Daten im Speicher

数据结构对齐的定义(data alignment)

  • 一个数据是和n-字节限制对齐的, 如果他的地址A是n的整数倍: A mod n = 0

 

常见的要求:

  • s-Byte的数据形式必须和s-字节限制对齐

  • 背景是: 储存器是用字(Worten)来组织的

    • 根据数据宽度(总是和架构宽度一致)传输一次访问, 比如32 Bit或者64 Bit
    • 对对齐的对象来说数据访问次数最小

 

Nichtausgerichtete Daten (misaligned data)

在一些处理器里面是允许的

  • 半字, 字或者双字允许储存在任意字节地址

 

优点

  • 在混合不同的数据形式时可以无缝使用储存器(不存在padding(填充)操作)
  • 对老的ISA有更好的兼容性(Kompatibilität)

 

缺点

  • 需要额外的内存访问
  • 更高的硬件需求

Von-Neumann Konzept: Speicher

程序和数据在同一个储存器中储存

  • 如今经常分开管理

  • 可修改的/不可修改的储存区域(同时也被称作Princeton Architektur)

  • 程序和数据在不同的储存器里面(同时也被称作Harvard Architektur)
  • 程序和数据在一个储存器中, 但是内部成两个分储存器

 

结果: 机器可以改变程序和数据

  • 对操作系统很重要(读取程序)

  • Anpassung von Programmen beim Linken(不知道咋翻译, 意思一下)

  • 但是: 基本没有能自我修改的程序

Herausforderungen in Mehrbenutzersystemen

起初系统是只为一个用户设计的

  • 一个时间点只有一个程序
  • 没有网络和远端的访问

 

如今的系统大多经常是多用户系统

  • 很多不相关的程序在储存器中
  • 可能同时执行

 

先决条件

  • 两个程序之间的数据保护
  • 程序间切换的可能

这些都是操作系统的任务!

 

需要来自硬件的支持

  • 有优先级的空间, 在这个空间里面操作系统工作
  • 中断/错误指示
  • 储存器管理

 

Unterstützung von Privilegierter Ausführung

如今的计算机大多是多用户系统

  • 用户程序不应该被允许直接访问通用硬件资源和系统配置
  • 需要通过操作系统编码
  • 技术基础: 执行空间

 

系统空间(Systemmodus)

  • 对所有计算机部分的完全的(优先的)访问
  • 给操作系统的

 

用户空间(Benutzermodus)

  • 被限制的访问
  • 没有优先的指令比如输入和输出
  • 无法访问系统配置

 

现代的处理器还可能有多个空间

  • 大多是用户和操作系统无法看见的
  • 只能硬件访问

 

Betriebssystem-Unterstützung: Moduswechsel

从系统空间到用户空间

  • 优先指令, 通常: 改写控制寄存器
  • 操作系统把这个用在程序的开头

 

从用户空间到系统空间

  • 必须被监控
  • 不能再系统空间里执行任意代码

 

通过特殊的指令实现

  • 也被称作: 系统调用, 巴拉巴拉的
    • 确保回跳地址, 跳到操作系统中预定义好的地址, 转入系统空间
    • 回跳指令再转回用户空间
    • 对比: 操作系统的明确定义的子程序

  • 系统调用指令会引发一个特殊的例外

 

中断/例外也能导致切换到优先(系统)空间

 

ERA023-System-Befehle Interrupts/Exceptions/Traps

 

Unterbrechungen und Ausnahmen(Interrupts und Exceptions/Traps)

冯诺依曼规定了顺序的执行

  • 程序计数器再每一条指令执行完后加一
  • 有一些例外: 比如跳跃
  • 对大多程序的单一执行是足够了

 

现代的系统更加复杂

  • 多用户系统
  • 复杂的外部设备
  • 互联网连接
  • 交互性

 

附加的: 错误处理

  • 比如: 除以0或者访问一个不存在的储存器
  • 理想状态: 在程序中处理

 

于是我们就引入了中断(Interrupt)和例外/陷阱(Exceptions/Traps)

 

Unterbrechungen („Interrupts“)

允许处理器对外部的, 非同步的事件作出反应

  • 特殊的输入信号: 中断请求
  • 可以遮盖(Maskierung)(处理)终端或者忽视(Ignorierung)

 

步骤(流程):

  • 信号在指令循环的尾部被询问
    • 可能是具体的外部信号线

  • 如果有信号

    • 把程序计数器保存在栈中
    • 跳到之前在操作系统里预定义好的地址(Unterbrechungsbehandlungsroutine, 中断处理例程 interrupt handler)
    • 回跳到原先的执行位置的尾部

  • 对比: 调用子程序, 但是不同步

 

主要应用:

  • 输入/输出
  • 时间计数器的运行

 

Ausnahmen („Traps“ / „Exceptions“)

执行不允许的操作会导致例外

  • 英语翻译, 自己看看

 

一些例子:

  • 不被允许的指令
  • 算术错误, 比如除以0
  • 储存器访问错误
    • 主要储存器里面没有对应的地址
    • 错误的访问权限
    • 硬件问题
  • 在硬件里面没有实现的指令

 

共同之处:

  • 被程序里的操作引发

 

Ausnahmebehandlung („Trap“ / „Exception“)

处理器中的过程

  • 中断刚才执行的指令
  • 保证运行状态(特别是程序计数器)
  • 跳到系统空间之前预定义的地址去

 

„Exception“-Vektortabelle

  • 存储了对于不同种类的例外的处理例程

 

操作系统中对例外的可能的处理

  • 停止处理
  • 仿真操作?(Emulieren der Operation)
  • 给用户查错

 

如果好有意义的话: 重复或者继续执行被中断的指令

 

Interrupts vs. Exceptions/Traps

两者都导致中断

  • 停止当前的程序
  • 储存系统状态(比如程序计数器)
  • 操作系统接过控制权
  • 结束后重返程序

 

中断是不同步的

  • 被外部事件触发
  • 可以在任意指令后发生
  • 中断大多情况下都是正常的运行

 

例外/陷阱是同步的

  • 在加工指令的时候触发
  • 在这条指令后发生
  • 通常情况下是执行中错误的信号

一个函数$f\left(x_{1} \ldots x_n\right)$中的Minterm $m$在DNF中叫蕴涵项, 如果在DNF中不存在一项$m’$, $m$和$m’$能够继续融合.

 

最小化的步骤

  1. 对每个Minterm(Einspunkt)找到最大的包含他的融合集合(他的蕴涵项)
  2. 找到蕴涵项集合中最小的包括所有Minterm的子集
  3. 这个子集中所有的蕴涵项的交集所描述的一些单项式就是最小的交集多项式(DMF-Disjunktive Minimalform)

 

Bemerkungen

在步骤1中不可融合的Minterm额外地每个都是蕴涵项

  • 结果不一定是单一的

 

  • 这也不一定包括最大的融合集合

Schaltwerk

时序逻辑电路(暂译)的例子:

  • 寄存器
  • 储存器
  • 控制部分(Leitwerk)

 

Schaltwerke (“sequential network”)

功能单元:

  • $m$个二进制输入: $E=e_{1}, \ldots, e_{m}$
  • $r$个二进制输出: $A=a_{1}, \ldots, a_{r}$

 

对二进制输入中的至少一个值:

  • 存在至少一个和这个值有关系的$A$中的值
  • 在这种情况下: $A$和到目前为止的$E$中的值有关

 

“内部”的大小($Q=q_{1}, \ldots, q_{n}$)

  • 时序逻辑电路的状态
  • 大小是$n$
  • $q_i$也是二进制的, $i \in(1, \ldots, n)$
  • 由到目前为止的$E$和$A$中的队列确定
  • 由$E$和$A$中的值确定

 

过程函数确定了$A$和$Q$

 

Konzept eines Schaltwerkes

自动机的结果和状态函数是一些组合逻辑电路

  • $\lambda$是负责输出逻辑的电路
  • $\delta$是负责内部状态的逻辑电路
  • 额外的输入: 回溯

 

理想状态下:

  • 每个在$\delta$的回溯中的延迟通过$\tau$来表示
  • 其他的路径都是瞬时的

 

那对于时间点$t_i$, 过程函数就可以这么给出:

  • $a^{t_{i}}=\lambda\left(e^{t_{i}}, q^{t_{i-\tau}}\right)$
  • $q^{t_{i}}=\delta\left(e^{t_{i}}, q^{t_{i-\tau}}\right)$

 

Beispiel: RS Latch

实现了状态的储存

  • 设置Bit: $s=1$
  • 还原Bit: $r=1$
  • 储存状态: $r/s=0$

 

电路的例子:

  • S把NOT(X)设置成0
  • R把X设置成0
  • 得出结果, 如果S=0并且R=0, 那就保持不变
  • 组合逻辑电路$\lambda$和$\delta$是交叉的

 

延迟$\tau$是已经通过电路技术给出了的

  • 但是也是几ns的一瞬间
  • 通常是不可计算的

Konzept der Parallelität

过程(Prozess)

  • 过程的定义: 一个过程是一个时间上可分解的行为. 一个分解把过程拆分成一些步骤.
  • 例子:
    • 程序实现
    • 组合逻辑电路中的信号流的推进

 

并行(Parallelität)

  • 并行的定义: 两个过程时并行的, 如果这两个过程开始之后至少有一个时间点他们不是独立运行的.
  • 例子:
    • 学期中的Vorlesung
    • 机器中信号的传递

 

Asynchronität, Synchronität, Synchronisation

不同步的过程

  • 不同步的过程的定义: 两个过程$P_1$和$P_2$是不同步的, 如果这两个过程之间没有固定的时间关系, 比如$P_!$不会在$P_2$步骤$j$开始或者结束之前开始步骤$i$, 或者反过来.
  • 例子:
    • 两个不相关联的电脑上运行两个程序
    • 两个没有链接的导线之中的信号

 

同步的过程: 例子

  • 火车(换乘链接)
  • 两个过程之间的配合

 

同步化

  • 一些操作使两个过程同步

功能性的定义:

一个计算是有功能的, 如果他把输入单一的, 可浮现的映射到输出上

 

例子:

  • 一序列的无交流的程序调用
  • 组合逻辑电路和组合逻辑电路的连接

 

不同步的有交流的一些过程总体上是没有功能性的, 即使每个单一的过程是功能性的

电路可以很简单地通过电厂效应晶体管实现(快, 小, 便宜)

Physikalische Schicht

  • 抽象的, 通过符号和特性描述的电子元件通过半导体(硅)实现

  • 半导体: 传导性位于导体和绝缘体之间

  • 有目的的混用参杂(p-或者n-Dotierung)让导体能在低温条件下工作

  • 参杂和电场在一个特定的几何排布下实现了晶体管

Transistoren als Schalter

重要的微电子的电路元素

  • 晶体管效应的原因是半导体(硅)的一个特性. 在一个n和p导体的接触区域产生一个阻断层
  • 电流只能从一个方向流过

每个晶体管都由两个这种pn通道组成, 这样就可以实现通常情况下没有电流能从晶体管流过

一个控制电流或者一个控制电压可以抵消这个阻断层, 让电流能从新流过
